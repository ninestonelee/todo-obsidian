# 🚀 Vercel 배포 가이드북

## 1. Vercel(버셀)이란?
Vercel은 우리가 만든 웹사이트를 인터넷 세상에 24시간 띄워주는 **'무료 호스팅(웹사이트 전시장)'** 서비스입니다.
마치 내 컴퓨터에만 있던 도화지(코드)를 전 세계 사람들이 볼 수 있는 갤러리(인터넷)에 걸어두는 것과 같습니다. 특히 프론트엔드 프로젝트(React, Next.js, 일반 HTML/CSS/JS 등)를 아주 쉽고 빠르게 배포할 수 있다는 강력한 장점이 있습니다.

---

## 2. 📋 사전 준비사항
배포를 시작하기 전에 아래 세 가지가 필요합니다.
1. **완성된 웹 프로젝트**: 내 컴퓨터에서 오류 없이 잘 실행되는 프로젝트 폴더.
2. **[GitHub(깃허브)](https://github.com/) 계정**: 내 코드를 안전하게 보관해 둘 온라인 저장소.
3. **[Vercel(버셀)](https://vercel.com/) 계정**: 로그인을 할 때 가급적 **GitHub 계정으로 연동해서 가입**하는 것을 강력히 추천합니다.

---

## 3. 🛠️ 단계별 배포 과정

### Step 1: 내 코드를 GitHub에 올리기 (가장 중요!)
Vercel은 내 컴퓨터의 코드를 직접 가져가지 않습니다. 대신 **GitHub에 올라간 코드를 복사해서** 안전하게 배포합니다.
1. GitHub 웹사이트에 로그인하여 새로운 저장소(Repository)를 만듭니다.
2. 내 컴퓨터의 프로젝트 코드를 해당 GitHub 저장소로 업로드(Push) 합니다.
   *(VS Code의 소스 제어(Source Control) 탭을 사용하거나 터미널 명령어를 사용하면 됩니다.)*

### Step 2: Vercel에 로그인하고 새 프로젝트 만들기
1. [Vercel 사이트](https://vercel.com/)에 접속하여 **[Log In]** 버튼을 누릅니다.
2. **'Continue with GitHub'** 버튼을 선택해 앞서 준비한 깃허브 계정으로 로그인합니다.
3. 로그인 후 보이는 대시보드 화면 우측 상단의 검은색 버튼 **[ Add New... ]** 를 누르고 **[ Project ]** 를 클릭합니다.

### Step 3: GitHub 저장소(Repository) 연결하기
1. 'Import Git Repository' 화면 좌측 리스트에 내 GitHub 저장소 목록이 자동으로 나타납니다.
2. 방금 코드를 업로드해 둔 저장소 이름 옆에 있는 **[ Import ]** 버튼을 클릭합니다.
   *(팁: 만약 내 저장소가 보이지 않는다면, 리스트 하단의 `Adjust GitHub App Permissions` 링크를 눌러 Vercel이 내 GitHub 저장소를 볼 수 있도록 권한을 전부 허용해주세요.)*

### Step 4: 배포 설정 및 배포 시작!
1. **Configure Project(프로젝트 설정)** 화면이 나타납니다.
   - **Project Name**: 웹사이트 주소에 쓰일 이름입니다. 원하는 이름으로 자유롭게 적어줍니다.
   - **Framework Preset**: Vercel이 아주 똑똑해서 내가 사용한 기술(React, Vue, Next.js 등)을 자동 인식합니다. (특별한 이유가 없다면 그대로 둡니다.)
   - **Environment Variables(환경 변수)**: API 키나 비밀번호처럼 외부로 노출되면 안 되는 값이 있다면 여기서 `Name`과 `Value` 빈칸에 입력하고 **[ Add ]**를 누릅니다.
2. 모든 확인이 끝났다면 파란색 **[ Deploy ]** 버튼을 클릭합니다.
3. 화면에 멋진 그래픽(축하 꽃가루 🎉)이 나타날 때까지 1~2분 정도 느긋하게 기다려줍니다.

---

## 4. ✨ Vercel의 마법: 완전 자동 배포 (CI/CD)
성공적으로 배포가 되셨나요? Vercel의 가장 멋진 부분은 바로 **'자동 업데이트'** 기능입니다.
사이트 내용을 수정하고 싶을 때 다시 Vercel 사이트에 들어올 필요가 없습니다. 

내 컴퓨터에서 코드를 고친 뒤 **GitHub에 다시 업로드(Push)만 하면**, Vercel이 알아서 코드가 변형된 것을 눈치채고 즉시 새로운 버전을 반영하여 내 웹사이트를 자동으로 업데이트 해줍니다!

## 5. 💡 자주 묻는 질문 (FAQ)

**Q. 제 웹사이트의 인터넷 주소(도메인)는 어떻게 되나요?**
A. 기본적으로 `(내가 정한 프로젝트 이름).vercel.app` 이라는 깔끔한 무료 링크를 제공해 줍니다. 이 링크를 지인들에게 보내면 폰으로도 접속할 수 있어요. 물론 나중에 개인 도메인(예: mydomain.com)을 직접 구매해서 연결하는 것도 가능합니다.

**Q. 배포 중에 빨간색 에러(Build Error)가 났어요!**
A. 내 컴퓨터에서는 완벽했는데 Vercel에서 에러가 난다면, 십중팔구 **대소문자 오타(파일명)**나 패키지가 덜 설치된 탓일 확률이 높습니다. (Mac과 Vercel 서버의 대소문자 인식 차이 때문). 에러 로그를 읽어보고 의심되는 파일 이름을 점검한 뒤 GitHub에 다시 올려보세요!

## 🤖 6. AI 에이전트(Claude Code, Cursor 등) 연동을 위한 안내

다른 AI 에이전트(Claude Code, Cursor 코덱스 등)가 Vercel 환경을 파악하고 터미널에서 제어할 수 있도록 돕는 연동 가이드입니다. 에이전트가 Vercel CLI를 사용해 직접 배포하거나 설정을 변경하게 만들려면 아래의 인증 정보가 필요합니다.

### 🔑 필요한 환경 변수 (Environment Variables)
AI 에이전트가 `vercel` 명령어를 비대화형(자동화)으로 실행하려면 아래 환경 변수들을 프로젝트 최상단의 `.env` (또는 `.env.local`) 파일에 저장하거나, 에이전트 세션의 환경 변수로 직접 넣어주어야 합니다.

1. `VERCEL_TOKEN`: Vercel 계정 [Tokens 페이지](https://vercel.com/account/tokens)에서 생성한 Access Token. (가장 중요)
2. `VERCEL_ORG_ID`: 연동할 Vercel 워크스페이스(조직/개인) ID. (Settings > General 에서 확인)
3. `VERCEL_PROJECT_ID`: 제어할 특정 프로젝트의 ID. (프로젝트 Settings > General 에서 확인)

### 💬 에이전트에게 전달할 프롬프트(지시어) 예시

에이전트에게 작업을 맡길 때 아래와 같이 명확하게 지시하시면 됩니다.

> **프롬프트 예시:**
> "현재 프로젝트 패렉토리에 있는 `.env` 파일 안에 `VERCEL_TOKEN`을 설정해 두었어.
> 이를 활용해서 `npx vercel --prod --token=$VERCEL_TOKEN --yes` 명령어를 실행해 이 프로젝트를 Vercel 프로덕션에 배포해줘."

### ⚠️ 주의사항
- **보안**: `VERCEL_TOKEN`은 절대로 GitHub(퍼블릭 버퍼)에 그대로 커밋(Push)하면 안 됩니다. 반드시 `.gitignore`에 `.env` 파일을 추가하여 로컬에만 보관하세요.
- 만약 에이전트가 `vercel` 명령어를 찾지 못한다면, `npx vercel [명령어]` 형태로 실행하도록 유도하는 것이 좋습니다.
